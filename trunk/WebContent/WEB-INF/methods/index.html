MAAB: datasets & methods

 

We describe the assembled transcriptomes produced for identification and analysis of the Hydroxyproline rich glycoprotein family (HRGP) and the methods for data analysis employed. 

 

 

Datasets

 

Each sample, over 1200 provided by the OneKP consortium, was assembled four times using a multiple-K methodology with K set to 39,49,59 and 69 consecutively. This leads to four transcriptomes per sample, one for each unique K-mer. Frequently, transcripts will be reconstructed by several k-mers. We make available each transcriptome using its four-letter OneKP-assigned sample ID:

Upon request, we can discuss large-scale data transfers but the full dataset is several terabytes (a terabyte is a million megabytes), so can take considerable time and effort to migrate. Note that all samples are

available with contamination, it is the user’s responsibility to consider results in the context of any sample contaminants which may be present.

Further information

 

Script to identify putative HRGP

This script, written using BioPerl and Perl, identifies the predicted proteins which are compositionally biased as expected to be part of the HRGP family. It performs steps 1b thru 1d and is given the raw assembly protein predictions

Script to extract putative ORFs from assemblies

This script, written using BioPerl and Perl, processes the Oases transcriptome assemblies and identifies putative ORFs translated into amino acids. It requires emboss getorf to be in the user’s PATH perform the translation.

Script to extract blast hit query sequences

Simple-minded perl script to extract those query sequences from a fasta file which are reported in the given blast hits (TSV output from blast is required for this script to work)

Supplementary material

Final processed hits from MAAB. All supplementary tables and figures as a single ZIP file.

Method to identify proline-rich regions

Describes the simple algorithm to identify proline-rich regions-of-interest (RoI) in protein sequence. We cite this URL in the paper rather than the long-winded explanation itself ;-)
