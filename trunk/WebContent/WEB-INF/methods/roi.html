MAAB: algorithm for identification of proline-rich regions (RoI)

 

During the course of the HRGP analysis, it became clear that identification of proline rich regions would be a key metric to aid in classification of the HRGP family members and rejection of some biased proteins. We term the regions of a protein sequence with biased residue composition, regions of interest (RoI). The percentage of the sequence covered by such biased regions is designated RoI% as a fraction of the total sequence length ie. including all signal sequence - where available.

 

The algorithm for the identification of RoI is described here. Although it is currently implemented in Java as part of the KNIME platform, it is a simple algorithm easily ported to other environments.

 

The algorithm proceed by processing each sequence independently: no state is preserved across sequences - determination of proline rich regions is based solely on primary sequence alone.

 

Preliminaries

 

The primary sequence is processed from left to right (N-terminal to C-terminal) using a window size of 8 and step of 1AA. 8 was chosen as the initial window, based on empirical observation from test sequence. At each step we evaluate the current window to determine:

1. If a proline-rich region has commenced. In this case, we keep extending the window until the bias falls below a priori determined thresholds. The thresholds were determined based on experimental, manual validation (data not shown) of test sequences

2. If a current proline-rich region should be extended

We require the current window to maintain at least 20% proline, and one of 70% PAST, 70% PSYK or 70% PVK. Note that this calculation is independent of all other calculations done by MAAB to each sequence: it does not depend on the number of motifs identified, signal sequence or anything else. We compute five columns for each sequence:

1. A visual, HTML, cell useful for visualising proline-rich regions (shown in BLUE within the KNIME table)

2. A percentage RoI% metric calculated as the total number of AA within RoI’s versus the total protein length

3. The number of proline rich regions

4. The proline-rich polypeptide sequences (as a collection)

5. The length of each RoI (in AA, as a collection)

 

These are used by MAAB to assist in classifying HRGPs (based on location and number of regions).

 

Core Algorithm

 

 

 

The core algorithm accumulates proline-rich windows in a bit vector (a set bit denotes a proline-rich region, clear bit not) of length set to the number of AA in the protein sequence. This bit vector is named “bv”.  Within the KNIME platform, two options are available:

l To report all RoI

l Report only those results belonging to proteins with at least 10% RoI

 

The function getHTMLCell() reports primary sequence colour-coded with RoI shown in blue, other regions in black. The implementation provides all the additional cells to the input table for the KNIME node.
